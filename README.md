# نظام مبتكر لإدارة الفواتير والمصاريف

نظام ويب كامل (فواتير + مصاريف + تقارير يومية/أسبوعية/شهرية) يتزامن تلقائيًا
بين أي جهاز/متصفح تفتح منه الرابط، عن طريق Firebase Firestore.

---

## 1) إعداد Firebase (خمس دقائق)

1. روح إلى https://console.firebase.google.com وسجّل دخول بنفس حسابك.
2. اضغط **"Add project" / إضافة مشروع** (أو استخدم مشروع موجود عندك).
3. من القائمة الجانبية اضغط **Build → Firestore Database**.
4. اضغط **Create database**، اختر **Start in production mode**، واختر أقرب
   منطقة (مثلاً `eur3` أو `me-central1` إذا متوفرة) ثم أنشئ.
5. روح إلى **⚙️ Project settings** (الترس بجانب "Project Overview") → تبويب
   **General** → انزل لقسم **"Your apps"** → اضغط أيقونة الويب `</>`.
6. سمّي التطبيق أي اسم (مثلاً "mobtaker-web") واضغط تسجيل. راح يعطيك كائن
   `firebaseConfig` فيه القيم (apiKey, authDomain, projectId...).
7. افتح ملف `firebase-config.js` في المشروع، واستبدل القيم بنفس القيم اللي
   نسختها من Firebase بالضبط.

### قواعد الأمان (Firestore Rules)
افتح **Firestore Database → Rules** والصق التالي (يسمح بالقراءة والكتابة
لأي شخص عنده رابط المشروع — مناسب لمشروع فردي بسيط، لكنه غير مخفي عن أي
حد يعرف رابط Firebase نفسه):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

> ⚠️ إذا حبيت لاحقًا تحمي البيانات بكلمة مرور/تسجيل دخول، أخبرني وأضيف لك
> Firebase Authentication (تسجيل دخول بالبريد) وأربطه بالقواعد.

اضغط **Publish** بعد اللصق.

---

## 2) رفع المشروع على GitHub ونشره كرابط مباشر (GitHub Pages)

1. روح إلى https://github.com وسجّل دخول → اضغط **New repository**.
2. سمّي المستودع مثلاً `mobtaker-invoices` واجعله **Public** → Create repository.
3. في صفحة المستودع الجديد اضغط **"uploading an existing file"** وارفع
   كل ملفات هذا المشروع دفعة وحدة:
   - `index.html`
   - `style.css`
   - `app.js`
   - `firebase-config.js` (بعد ما تعبيه ببياناتك)
   - `README.md`
4. اكتب أي رسالة Commit واضغط **Commit changes**.
5. روح إلى **Settings** (داخل المستودع) → تبويب **Pages** من القائمة
   الجانبية.
6. تحت **Build and deployment → Source** اختر **Deploy from a branch**،
   وتحت **Branch** اختر `main` والمجلد `/ (root)` → Save.
7. انتظر دقيقة، وراح يظهر لك رابط شبيه بـ:
   `https://اسم_حسابك.github.io/mobtaker-invoices/`
8. هذا هو رابط نظامك الدائم — افتحه من أي جهاز أو متصفح وراح تكون كل
   البيانات (الفواتير، المصاريف، رقم الفاتورة، الشعار) متزامنة تلقائيًا
   لأنها كلها متخزنة في Firestore وليس في المتصفح.

---

## 3) طريقة الاستخدام

- **لوحة التحكم**: تقارير فورية لليوم / آخر 7 أيام / هذا الشهر (عدد
  الفواتير، الدخل، المصاريف، الربح الصافي).
- **فاتورة جديدة**: عبّي البيانات واضغط **حفظ الفاتورة** — رقم الفاتورة
  يُحجز تلقائيًا من عداد مشترك على Firebase، فما يتكرر أبدًا حتى لو
  فتحت النظام من أكثر من جهاز بنفس الوقت. بعدها اضغط **طباعة / PDF**.
- **الفواتير**: سجل بكل الفواتير المحفوظة.
- **المصاريف**: أضف مصروف (خامات، صيانة، شحن...) وشوف السجل.
- **الشعار**: يُرفع مرة وحدة ويُخزَّن في Firebase، فيظهر تلقائيًا في كل
  الأجهزة بدون ما ترفعه من جديد.

---

## ملاحظات

- إذا رجّعت خطأ عند الحفظ، تأكد إن `firebase-config.js` معبّى صح وإن
  Firestore Rules منشورة كما بالأعلى.
- التقارير الأسبوعية محسوبة كـ "آخر 7 أيام" (متحركة) وليس أسبوع تقويمي
  ثابت، عشان تعطيك صورة دقيقة أي وقت فتحت النظام.
- تقدر تضيف لاحقًا: تسجيل دخول، نسخ احتياطي، تصدير Excel، إشعارات واتساب
  تلقائية — أخبرني إذا حبيت أي منها.
